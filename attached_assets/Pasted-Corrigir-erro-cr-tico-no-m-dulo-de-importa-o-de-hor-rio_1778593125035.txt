Corrigir erro crítico no módulo de importação de horários escolares.

Problema identificado:

Atualmente, sempre que o usuário importa um horário de um turno, o sistema remove ou sobrescreve o turno já existente.

Exemplo:

* ao importar o horário matutino, o vespertino desaparece;
* ao importar o vespertino, o matutino desaparece.

Ou seja:
os turnos não estão sendo armazenados separadamente.

Isso está causando:

* perda de dados;
* necessidade de reimportação;
* retrabalho constante;
* inconsistência nos horários escolares.

Correções necessárias:

1. Armazenamento Separado por Turno:
   O sistema deve armazenar:

* Matutino;
* Vespertino;
* Noturno;

em estruturas independentes.

Exemplo:

```json id="nprw2m"
{
  "matutino": {},
  "vespertino": {},
  "noturno": {}
}
```

2. Não Sobrescrever Dados:
   Ao importar um novo arquivo:

* adicionar o novo turno;
* manter os turnos já existentes;
* evitar limpeza automática do estado anterior.

3. Mesclagem Inteligente:
   Implementar merge automático dos dados importados.

Exemplo:

* importar manhã;
* depois importar tarde;

resultado:

* ambos os turnos permanecem visíveis simultaneamente.

4. Persistência:
   Após:

* salvar;
* atualizar a página;
* reiniciar o sistema;

todos os turnos devem continuar disponíveis.

5. Verificar Problemas:
   Analisar possíveis falhas em:

* LocalStorage;
* IndexedDB;
* gerenciamento de estados;
* sobrescrita de arrays;
* reset de objetos;
* importação Excel;
* renderização dos turnos.

6. Interface:
   Mostrar simultaneamente:

* Matutino;
* Vespertino;
* Noturno.

Sem:

* ocultar;
* substituir;
* desabilitar turnos.

7. Importação:
   Aceitar corretamente:

* múltiplos arquivos;
* importações sequenciais;
* importação de vários turnos.

8. Objetivo Final:
   O sistema deve permitir importar horários de vários turnos sem perder os dados já cadastrados, mantendo todos os horários ativos simultaneamente.
