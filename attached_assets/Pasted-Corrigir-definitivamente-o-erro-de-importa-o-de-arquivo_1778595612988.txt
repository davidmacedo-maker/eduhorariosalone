Corrigir definitivamente o erro de importação de arquivos Excel do sistema Cronos Horário.

Problema identificado:

Ao importar arquivos Excel gerados pelo sistema:

```text id="q6l2sv"
Cronos Horário
```

o sistema continua apresentando o seguinte erro:

* ao importar o horário matutino, o horário vespertino desaparece;
* ao importar o horário vespertino, o horário matutino desaparece;
* os turnos estão sendo sobrescritos;
* apenas um turno permanece carregado por vez.

Isso indica que o sistema:

* está substituindo o estado anterior;
* não está armazenando os turnos separadamente;
* está limpando os dados já importados.

Correções obrigatórias:

1. Estrutura Separada por Turno:
   Criar armazenamento independente para:

* Matutino;
* Vespertino;
* Noturno.

Exemplo:

```json id="5u6j7y"
{
  "matutino": {},
  "vespertino": {},
  "noturno": {}
}
```

2. Não Sobrescrever Dados:
   Ao importar um novo arquivo:

* NÃO apagar os turnos existentes;
* NÃO resetar o estado;
* NÃO limpar LocalStorage/IndexedDB;
* NÃO substituir arrays completos.

3. Merge Inteligente:
   Implementar:

```text id="t6f4o3"
merge de dados
```

Ao importar:

* manhã;
* depois tarde;

resultado esperado:

* ambos os turnos permanecem ativos simultaneamente.

4. Compatibilidade com Cronos Horário:
   Analisar especificamente a estrutura do Excel exportado pelo:

```text id="1knxtc"
Cronos Horário
```

Identificar:

* abas;
* colunas;
* horários;
* turnos;
* códigos internos.

Adaptar o parser/importador para essa estrutura.

5. Persistência:
   Após:

* importar;
* salvar;
* atualizar página;
* fechar o sistema;

todos os turnos devem continuar carregados.

6. Correção do Estado:
   Verificar problemas relacionados a:

* React state;
* sobrescrita de objetos;
* setState incorreto;
* arrays substituídos;
* reducers;
* contexto global;
* sincronização do armazenamento.

7. Interface:
   Mostrar simultaneamente:

* Matutino;
* Vespertino;
* Noturno.

Sem:

* ocultar;
* desativar;
* substituir turnos.

8. Teste Obrigatório:
   Executar testes reais importando:

* um arquivo matutino;
* um arquivo vespertino;
* ambos em sequência.

Confirmar que:

* nenhum turno desapareça;
* os dados coexistam corretamente.

9. Objetivo Final:
   O sistema deve permitir importar múltiplos horários do Cronos Horário sem perda de dados, mantendo todos os turnos ativos simultaneamente.
